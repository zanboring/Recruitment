## 二、逐项修改详情

### 修改 1: DataManage.vue（第 342-343 行）

**问题**：`http.post('/crawl/task', task)` 的返回值直接被 `as number` 断言，但 http 拦截器返回的是 `res.data`（已 unwrap），后端可能返回 `{ id: 123 }` 对象而非纯数字。

**修改前**：
```typescript
const taskId = await http.post('/crawl/task', task) as number;
```

**修改后**：
```typescript
const taskRes = await http.post('/crawl/task', task);
const taskId = typeof taskRes === 'number' ? taskRes : (taskRes as any).id || taskRes;
```

#### 2b. handleJobClick 增强点击跳转逻辑（第 303-314 行）

**修改前**：
```typescript
const handleJobClick = (job: any) => {
  if (job.url) { window.open(job.url, '_blank'); }
  else { ElMessage.warning('该岗位暂无详情链接'); }
};
```

**修改后**：
```typescript
const handleJobClick = (job: any) => {
  console.log('[JobCard clicked]', job.id, job.title, 'url:', job.url);
  if (job.url && job.url !== '' && !job.url.includes('baidu.com')) {
    window.open(job.url, '_blank', 'noopener,noreferrer');
  } else if (job.url) {
    // 百度备用链接或其他链接，仍然尝试跳转
    window.open(job.url, '_blank', 'noopener,noreferrer');
  } else {
    // 完全没有URL时给出提示
    ElMessage.warning('该岗位暂无详情链接');
  }
};
```

#### 2c. blob 导出修复（第 213-215 行）

**问题**：http 拦截器对 blob 请求返回 Blob 本身，但原代码取 `.data` 得到 undefined。

**修改前**：
```typescript
const blob = new Blob([response.data], { type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet' });
const url = window.URL.createObjectURL(blob);
```

**修改后**：
```typescript
// 拦截器对 blob 请求直接返回 Blob 对象，无需再取 .data
const blob = response instanceof Blob ? response : (response as any).data;
const blobData = blob instanceof Blob ? blob : new Blob([blob], { type: 'application/vnd.openxmlformats-officedocument.spreadsheetml.sheet' });
const url = window.URL.createObjectURL(blobData);
```

### 修改 3: JobCard.vue（卡片点击事件修复）

**问题**：el-card 组件的 click 事件可能被内部拦截或事件冒泡异常，导致点击卡片无反应。

**模板部分修改前**：
```html
<template>
  <el-card class="job-card" shadow="hover">
    <!-- ... 内容 ... -->
  </el-card>
</template>
```

**模板部分修改后**：
```html
<template>
  <div class="job-card-wrapper" @click="$emit('click', job)">
  <el-card class="job-card" shadow="hover">
    <!-- ... 内容 ... -->
  </el-card>
  </div>
</template>
```

**CSS 部分修改前**：
```css
.job-card {
  height: 100%; border: none; border-radius: 16px; background: #fff;
  transition: all .4s cubic-bezier(.4, 0, .2, 1); overflow: hidden; cursor: pointer;
}
.job-card:hover { transform: translateY(-8px); box-shadow: 0 16px 40px rgba(64,158,255,.15) !important; }
```

**CSS 部分修改后**：
```css
.job-card-wrapper {
  cursor: pointer;
  height: 100%;
}
.job-card {
  height: 100%; border: none; border-radius: 16px; background: #fff;
  transition: all .4s cubic-bezier(.4, 0, .2, 1); overflow: hidden;
}
.job-card-wrapper:hover .job-card { transform: translateY(-8px); box-shadow: 0 16px 40px rgba(64,158,255,.15) !important; }
```

> **核心改动**：在外层包裹一个原生 `<div>` 作为事件绑定层，确保 click 事件可靠触发。

### 修改 7: markdownRenderer.ts（移除 marked-gfm 依赖，第 8-12 行）

**问题**：`marked-gfm` 未在 package.json 中声明依赖，Vite 动态 import 解析失败。

**修改前**：
```typescript
import { gfm } from 'marked-gfm';
import { marked } from 'marked';
// ...
marked.use(gfm());
```

**修改后**：
```typescript
import { marked } from 'marked';

// 启用 GFM（GitHub Flavored Markdown）语法支持
// marked v12+ 内置 GFM，无需额外安装 marked-gfm
marked.use(marked.gfm ? marked.gfm() : {});
```
