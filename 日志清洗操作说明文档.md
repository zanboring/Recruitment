# 日志数据清洗操作说明文档

## 1. 清洗操作概述

本次清洗操作针对 `recruitment-system` 项目的日志文件，执行了全面的数据清洗流程，包括无效记录去除、格式标准化、异常数据过滤等操作，确保日志数据的完整性、一致性和准确性。

## 2. 清洗步骤详细说明

### 2.1 去除无效记录

**处理规则**：
- 识别并移除格式不完整的日志条目
- 移除无法解析的日志记录
- 移除无实际业务意义的空行或格式错误的记录

**判断标准**：
- 日志格式是否匹配标准格式：`YYYY-MM-DD HH:MM:SS.SSS [thread] LEVEL logger - message`
- 时间戳格式是否正确
- 日志级别是否在有效范围内

**实施方法**：
使用正则表达式匹配标准日志格式，对不符合格式的记录标记为无效并移除。

### 2.2 处理缺失值

**处理策略**：
- 对于时间戳缺失：直接标记为无效记录
- 对于线程信息缺失：保留记录，使用空值填充
- 对于日志级别缺失：直接标记为无效记录
- 对于 logger 信息缺失：保留记录，使用空值填充
- 对于 message 缺失：保留记录，使用空值填充

**实施方法**：
通过正则表达式分组捕获，检查各字段是否存在，对缺失字段采取相应处理策略。

### 2.3 纠正格式错误

**处理规则**：
- 统一时间戳格式为 `YYYY-MM-DD HH:MM:SS`
- 规范日志级别标识（DEBUG、INFO、WARN、ERROR、FATAL）
- 修正字段分隔符使用不一致问题

**实施方法**：
- 使用 datetime 模块解析并重新格式化时间戳
- 验证日志级别是否在有效范围内
- 确保字段分隔符使用一致（空格分隔）

### 2.4 过滤异常数据

**处理规则**：
- 设定合理阈值范围，识别并处理超出正常业务范围的异常值
- 识别并移除重复记录
- 处理明显不合逻辑的日志内容

**判断标准**：
- 重复记录：通过计算记录哈希值进行检测
- 异常时间戳：检查时间戳是否在合理范围内
- 异常日志级别：确保日志级别在有效范围内

**实施方法**：
- 使用集合存储记录哈希值，检测并移除重复记录
- 验证时间戳格式和范围
- 验证日志级别有效性

### 2.5 标准化数据格式

**处理规则**：
- 统一字段命名规范
- 确保同类信息采用一致的数据表示方式
- 建立日志数据标准模板

**实施方法**：
- 确保所有日志记录遵循统一的格式模板
- 标准化时间戳、线程信息、日志级别等字段的表示方式
- 保持字段顺序和格式的一致性

## 6. 发现的重大数据质量问题及处理建议

### 6.1 重复记录问题
- **问题**：存在大量重复的日志记录，占总记录数的 29.31%
- **原因**：可能是由于系统重复执行相同的操作或日志记录机制问题
- **建议**：
  - 检查系统代码，避免重复执行相同操作
  - 优化日志记录机制，避免重复记录
  - 定期执行日志清洗，移除重复记录

### 6.2 格式不一致问题
- **问题**：部分日志文件存在格式不一致的问题，导致大量记录被标记为无效
- **原因**：可能是由于日志配置变更或不同组件使用不同的日志格式
- **建议**：
  - 统一系统日志配置
  - 确保所有组件使用相同的日志格式
  - 建立日志格式验证机制

### 6.3 备份文件处理
- **问题**：`recruitment-system.log.backup` 文件中的所有记录都被标记为无效
- **原因**：备份文件可能包含了非标准格式的内容或格式错误
- **建议**：
  - 检查备份文件的生成机制
  - 确保备份文件格式与原始日志文件一致
  - 定期清理过期的备份文件
